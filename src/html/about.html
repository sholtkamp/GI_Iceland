DisasterMap

The application DisasterMap was implemented during the class Geoinformation in Society at Institute for Geoinformatics of the Westfälische-Wilhelms-Universität in the Wintersemester 2018/2019

DisasterMap is a modern, responsive web interface, providing data about volcanos in Iceland. It is designed to educate tourists and people living there about the risks and the intended behaviour in cases of emergency when a volcano eruption takes place. This should mitigate the effects of volcanic eruptions in Iceland.

Data

It makes use of linked data that was prepared in the Virtuoso TripleStore as RDF-Triples. Freely available data about Iceland prepared in shapefiles was converted to RDF-Triples to and uploaded to the TripleStore. By the use of existing vocabularies the new shapefile data was linked to other datasets. In total six different shapefiles were used containing: places (cities, villages, ...), waterways, the border, roads, volcanos and natural features.

Goals

The overall goal of the application is to mitigate the effects of volcanic eruptions in Iceland through providing information and educating the people about these natural disasters. This should raise awareness and start a more detailed thinking about volcanic eruptions by using freely available data and provide a meaningful visualization of it. By this the inhabitants might rethink their choice of a location for their new house or other infrastructural decisions.

Technologies

To convert the shapefiles to RDF-Triples the programming language R was used together with the R-package "rdflib". This enables the automatic creation of an RDF object and also the serialization of it to a turtle file. For the visualization the standard technologies of web-programming: HTML, CSS and JavaScript were used. For the design of the web application the JavaScript library Bootstrap was used. To get the interactive map the JavaScript library Leaflet was used which makes it possible to load the different datasets as layers on a base map. To create educational information for the users we used piktocharts.
